# 요구사항: 할 일 날짜 반복 (date-repeat)

> **다음 단계 안내**: 이 문서는 PRD (기획 단계 요구사항만) 입니다. 다음 단계로 `designing-direction` skill (또는 `/design` 슬래시) 을 호출해서 `date-repeat-tech-design.md` (기술 설계서) 를 만드세요. 기술 결정이나 구현 세부사항은 여기 박지 마세요.

## 1. 배경/목적

대표님은 "매월 1일 정산", "매주 월요일 주간보고", "매일 비타민" 처럼 **주기적으로 반복되는 할 일**을 매번 손으로 다시 만들어야 했다. 반복 규칙을 한 번만 정의해두면 정해진 날짜마다 자동으로 '오늘 할 일'에 나타나도록 하여 수동 재생성 부담을 없앤다.

핵심 가치: **반복 규칙 1회 정의 → 발생일마다 자동 노출**. 이 앱의 1순위 성공 기준인 가독성·가시성을 해치지 않도록, 고빈도 반복이 미체크로 누적될 때 화면이 도배되지 않게 한다.

## 2. 사용자 스토리 / 시나리오

- **(누가)** 1인 사용자(대표님) **(무엇을)** 할 일에 반복 규칙(매일/매주/매월/매년 + N간격)을 지정 **(왜)** 주기적 업무를 매번 다시 입력하지 않기 위해.
- **시나리오 A (정산형)**: '매월 1일 정산' 을 만들어두면 매달 1일 오늘 화면에 자동 등장. 1일에 못 하면 다음 날로 이월되어 계속 보인다 (놓침 방지).
- **시나리오 B (습관형)**: '매일 비타민' 을 만들어두면 매일 등장. 3일 빼먹어도 오늘 화면에는 **가장 오래된 미체크 1건만** 노출되고 나머지는 묶음/배지로 표시 (화면 도배 방지). 데이터상 누락 이력은 보존.
- **시나리오 C (종료)**: '2026-12-31 까지 매주 월요일 주간보고' 처럼 종료일을 지정하면 그 이후로는 생성되지 않는다.
- **시나리오 D (캘린더)**: 반복 할 일에 시간/날짜 틀을 채우면 Google Calendar 에 반복(RRULE) 일정으로 등록된다.

## 3. 기능 요구사항 (FR)

- **FR-1 (반복 규칙 지정)**: 할 일 추가/편집 시 반복 규칙을 설정할 수 있다. 지원 주기: **매일 / 매주(특정 요일) / 매월(특정 일) / 매년**, 각 주기에 **N간격**(예: 2주마다, 3일마다) 지정 가능.
- **FR-2 (자동 발생/노출)**: 반복 규칙에 따라 정해진 발생일이 되면 해당 할 일이 '오늘 할 일'에 자동으로 나타난다. 사용자가 매번 수동 생성하지 않는다.
- **FR-3 (미체크 이월)**: 반복 발생분을 그날 체크하지 않으면 일반 할 일과 동일하게 **다음 날로 이월**되어 계속 노출된다 (놓침 방지).
- **FR-4 (이월 누적 dedup 표시)**: 같은 반복 규칙의 미체크 항목이 2건 이상 쌓이면, '오늘 할 일' 화면에는 **가장 오래된 1건만** 노출하고 나머지는 **묶음(개수 배지 등)**으로 표시한다. 누적 데이터(각 발생분 이력)는 삭제하지 않고 보존한다.
- **FR-5 (반복 종료 조건)**: 반복은 기본 **무한**이며, 선택적으로 **종료일**을 지정할 수 있다. 종료일 이후 발생분은 생성되지 않는다.
- **FR-6 (개별 인스턴스 vs 규칙)**: 반복으로 생성된 개별 발생분의 체크/삭제는 그 발생분에만 적용되고, 반복 규칙 자체는 유지된다. 반복 규칙의 수정/해제/삭제는 별도 동작으로 제공한다.
- **FR-7 (반복 표시)**: 반복으로 생성된 할 일은 일반 할 일과 구분되는 시각 표식(반복 아이콘 등)을 가진다.
- **FR-8 (Google Calendar RRULE)**: 반복 할 일에 날짜/시간 틀이 있으면 Google Calendar 에 RRULE 반복 이벤트로 등록/수정/삭제된다.

## 4. 비기능 요구사항 (NFR)

- **NFR-1 (성능)**: 반복 인스턴스 생성/표시 로직이 추가돼도 '오늘 할 일' 재계산이 체감 지연(입력→반영 200ms 이내, 60fps) 을 깨지 않는다.
- **NFR-2 (동기화)**: 반복 규칙·생성된 인스턴스는 기존 Drift(로컬) + Supabase + Outbox 동기화 경로를 그대로 타며, macOS ↔ Android 5초 이내 반영을 유지한다.
- **NFR-3 (오프라인)**: 오프라인 상태에서도 발생일 도래 시 인스턴스가 로컬에서 정상 노출된다 (서버 의존 없이).
- **NFR-4 (가시성)**: FR-4 dedup 덕에 고빈도 반복 누적 시에도 '오늘 할 일' 화면 항목 수가 폭증하지 않는다.

## 5. 범위 밖 (Out of Scope)

- 복잡한 사용자 정의 RRULE 풀세트 (예: '매월 마지막 평일', '매월 둘째·넷째 화요일' 같은 고급 패턴) — 이번엔 표준 4주기 + N간격까지만.
- 반복 종료 조건 중 **N회 후 종료**(횟수 기반) — 이번엔 무한 + 종료일까지만 (대표님이 종료일 방식 선택).
- 반복 인스턴스별 개별 시간 변경/예외(이번 주만 화요일로 이동 같은 단일 예외 편집).
- 캘린더에서 역방향(Google → 앱)으로 반복 일정 가져오기.

## 6. 수용 기준 (Acceptance Criteria)

- **AC-1**: 매일/매주(요일)/매월(일)/매년 각각에 N간격을 지정한 반복 규칙을 만들 수 있고, 다음 발생일이 규칙대로 계산된다. (Yes/No)
- **AC-2**: 발생일이 되면 수동 조작 없이 해당 할 일이 '오늘 할 일'에 나타난다. (Yes/No)
- **AC-3**: 반복 발생분을 미체크로 두면 다음 날에도 노출된다 (이월). (Yes/No)
- **AC-4**: 같은 반복의 미체크가 2건 이상이면 '오늘 할 일'에 1건만 보이고 나머지는 묶음/배지로 표시되며, 데이터 이력은 보존된다. (Yes/No)
- **AC-5**: 종료일을 지정한 반복은 종료일 이후 발생분이 생기지 않는다. (Yes/No)
- **AC-6**: 반복으로 생성된 할 일이 시각 표식으로 구분된다. (Yes/No)
- **AC-7**: 반복 할 일이 Google Calendar 에 RRULE 이벤트로 등록되고, 규칙 수정/삭제가 캘린더에 반영된다. (Yes/No)
- **AC-8**: 반복 규칙·인스턴스가 macOS ↔ Android 간 동기화된다. (Yes/No)
