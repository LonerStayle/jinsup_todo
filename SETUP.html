Solo Todo — 첫 실행 SETUP 가이드

앱 코드 외부에서 한 번 셋업이 필요한 항목. 위에서 아래로 순서대로 진행해 주십시오. 체크박스를 하나씩 채우면서 진행하시면 빠짐없이 처리됩니다.

1 환경변수 .env.local

저장소 루트의 .env.example 을 .env.local 로 복사 후 4 키를 채우십시오. .env.local 은 .gitignore 에 포함되어 있어 커밋되지 않습니다.

키발급처비고
SUPABASE_URL Supabase Dashboard → Project Settings → API 예: https://abc.supabase.co
SUPABASE_ANON_KEY 동일 JWT (eyJ…)
GOOGLE_OAUTH_CLIENT_ID_DESKTOP Google Cloud Console → APIs & Services → Credentials Type: Desktop. macOS 빌드용
GOOGLE_OAUTH_CLIENT_ID_ANDROID 동일 Type: Android, package + SHA-1 fingerprint 필요

실행 시 주입

flutter run --dart-define-from-file=.env.local
flutter build macos --release --dart-define-from-file=.env.local
flutter build apk --release --dart-define-from-file=.env.local
  • Supabase URL + Anon Key 입력
  • Google OAuth Desktop client id 입력 (Calendar 안 쓸 거면 비워둬도 OK)
  • Google OAuth Android client id 입력

2 Supabase 프로젝트 + RLS

  1. Supabase Dashboard 에서 새 프로젝트 생성 (region: ap-northeast-2 서울 권장).
  2. 좌측 메뉴 Authentication → Providers → Email 활성화. "Magic Link" 사용.
  3. Database → SQL Editor 에서 아래 SQL 실행. 저장소에도 파일로 있습니다:
    • 신규 셋업: supabase/schema.sql
    • 이전 버전에서 옮길 때: supabase/migrate.sql
-- 별도 schema 'solo_todo' 안에 격리. 다른 앱이 같은 프로젝트의 public 을 쓰더라도 영향 X.
create schema if not exists solo_todo;

-- Supabase 의 PostgREST + Realtime 이 사용하는 role 에 schema 사용 권한 부여.
grant usage on schema solo_todo to anon, authenticated, service_role;
alter default privileges in schema solo_todo
  grant all on tables to anon, authenticated, service_role;
alter default privileges in schema solo_todo
  grant all on sequences to anon, authenticated, service_role;

-- todos 테이블
create table if not exists solo_todo.todos (
  id                text primary key,
  user_id           uuid references auth.users(id) on delete cascade not null,
  title             text not null,
  category          text not null,
  due_at            timestamptz,
  done_at           timestamptz,
  created_at        timestamptz not null default now(),
  updated_at        timestamptz not null default now(),
  calendar_event_id text
);

-- 이미 만든 테이블에도 grant (default privileges 는 새 객체에만 적용되므로 명시)
grant all on solo_todo.todos to anon, authenticated, service_role;

-- RLS — 본인 row 만
alter table solo_todo.todos enable row level security;
create policy "user_read_own"   on solo_todo.todos for select  using (auth.uid() = user_id);
create policy "user_insert_own" on solo_todo.todos for insert  with check (auth.uid() = user_id);
create policy "user_update_own" on solo_todo.todos for update  using (auth.uid() = user_id);
create policy "user_delete_own" on solo_todo.todos for delete  using (auth.uid() = user_id);

-- 동기화 정렬용 인덱스
create index todos_user_updated on solo_todo.todos (user_id, updated_at desc);

-- Realtime publication 에 추가 (Dashboard 의 Database → Replication 토글과 동등)
alter publication supabase_realtime add table solo_todo.todos;
Supabase Dashboard → Settings → API → Exposed schemas 에 solo_todo 를 반드시 추가해 주십시오. (default 는 public, graphql_public 만이라 새 schema 는 명시해 줘야 PostgREST 가 인식합니다.)

이미 이전 버전 SQL 을 실행했었다면 (마이그레이션)

예전에 public.todos 또는 public.solo_todo_todos 로 실행한 적이 있다면 아래 마이그레이션 SQL 을 추가 실행해 주십시오. 두 경우 모두 solo_todo.todos 로 이동합니다. 해당 테이블이 없으면 no-op 으로 안전.

-- (위 메인 SQL 의 'create schema if not exists solo_todo;' 가 이미 실행된 뒤에 호출하세요)

do $$
begin
  -- Case A: public.todos (가장 첫 버전)
  if exists (
    select 1 from information_schema.tables
    where table_schema = 'public' and table_name = 'todos'
  ) then
    alter table public.todos set schema solo_todo;
  end if;

  -- Case B: public.solo_todo_todos (prefix 버전)
  if exists (
    select 1 from information_schema.tables
    where table_schema = 'public' and table_name = 'solo_todo_todos'
  ) then
    alter table public.solo_todo_todos set schema solo_todo;
    alter table solo_todo.solo_todo_todos rename to todos;
  end if;
end $$;

-- 옮긴 객체들도 새 grant 부여
grant all on solo_todo.todos to anon, authenticated, service_role;

-- 옛 publication 매핑 정리 후 새 schema 의 테이블 등록 (이미 있으면 no-op)
alter publication supabase_realtime drop table if exists public.todos;
alter publication supabase_realtime drop table if exists public.solo_todo_todos;
alter publication supabase_realtime add  table solo_todo.todos;
  1. Settings → API → Exposed schemas 에 solo_todo 추가 (default public, graphql_public 옆에 콤마 + solo_todo).
  2. Authentication → URL Configuration → Redirect URLs 에 매직링크 콜백 URL 추가:
    • macOS: io.supabase.solo_todo://login-callback
    • Android: com.goldenplanet.solo_todo://login-callback
  • 프로젝트 생성
  • Email Magic Link 활성화
  • SQL 실행 (schema + todos 테이블 + RLS + Realtime add)
  • Exposed schemas 에 solo_todo 추가
  • Redirect URLs 두 개 등록

3 Google Cloud (Calendar API)

  1. console.cloud.google.com 에서 새 프로젝트 생성.
  2. APIs & Services → Library → "Google Calendar API" Enable.
  3. OAuth consent screen 설정 (External + 본인 이메일).
  4. Credentials → + Create credentials → OAuth client ID:
    • Desktop app 타입 → client ID 발급 → GOOGLE_OAUTH_CLIENT_ID_DESKTOP 에 입력
    • Android 타입 → package: com.goldenplanet.solo_todo + SHA-1 fingerprint 
      cd android && ./gradlew signingReport
      출력 중 SHA1 값을 복사. GOOGLE_OAUTH_CLIENT_ID_ANDROID 에 입력.
  5. Scope 는 https://www.googleapis.com/auth/calendar.events 한 개. 코드가 자동 요청.
  • Calendar API Enable
  • OAuth consent screen 설정
  • Desktop client ID 발급 + .env.local 입력
  • Android client ID 발급 + .env.local 입력

4 macOS 빌드 (.app)

Xcode 풀 설치 + CocoaPods 가 필요합니다. 라이트한 Command Line Tools 만으론 부족합니다. 
# 1) Xcode 풀 설치 (Mac App Store) + 첫 실행
sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -runFirstLaunch

# 2) CocoaPods
sudo gem install cocoapods
# 또는 Homebrew: brew install cocoapods

# 3) 빌드
flutter pub get
pod install --project-directory=macos
flutter build macos --release --dart-define-from-file=.env.local

산출물: build/macos/Build/Products/Release/solo_todo.app

Entitlements (참고)

App Sandbox + Network Client 권한이 이미 macos/Runner/Release.entitlements 에 적용돼 있습니다. Supabase / Google Calendar outbound HTTPS 자동 동작.

  • Xcode 풀 설치 + xcode-select --switch
  • CocoaPods 설치
  • flutter build macos --release 성공
  • .app 더블클릭 후 sign-in 동작 확인
  • 메뉴바 (tray) 아이콘 노출 확인
  • Cmd+N 글로벌 단축키 동작 확인 (앱 백그라운드에서도)

6 Deep Link (매직링크 콜백)

이메일 매직링크 클릭 후 OS 가 앱으로 되돌아오게 하려면 URL scheme 등록이 필요합니다.

macOS — macos/Runner/Info.plist

<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleURLSchemes</key>
    <array><string>io.supabase.solo_todo</string></array>
  </dict>
</array>

Android — android/app/src/main/AndroidManifest.xml

<activity ...>
  ...
  <intent-filter>
    <action android:name="android.intent.action.VIEW"/>
    <category android:name="android.intent.category.DEFAULT"/>
    <category android:name="android.intent.category.BROWSABLE"/>
    <data android:scheme="com.goldenplanet.solo_todo" android:host="login-callback"/>
  </intent-filter>
</activity>
  • macOS Info.plist 의 CFBundleURLTypes 등록
  • Android intent-filter 등록
  • Supabase Dashboard 의 Redirect URLs 와 일치하는지 확인

7 운영 메모 & 성능 확인

자주 보는 명령 — Makefile 한방

모든 dart-define / device 선택은 make 가 알아서. .env.local 자동 감지.

make help          # 사용법
make setup         # flutter pub get + (macOS) pod install
make codegen       # freezed/json/drift codegen

make run           # macOS 데스크탑 실행
make run-android   # Android 첫 device 자동 선택 실행

make build-macos   # release .app 산출
make build-apk     # release .apk 산출

make check         # analyze + format-check + test (커밋 직전)
make test          # flutter test
make format        # dart format .

make clean         # flutter clean + pub get

raw 명령으로 직접 실행하시려면:

flutter run -d macos --dart-define-from-file=.env.local
flutter devices
flutter run -d <device-id> --dart-define-from-file=.env.local
dart analyze && dart format --output=none --set-exit-if-changed . && flutter test

60fps 트레이스 (디자인 점수 확인용)

release 빌드로 1 분 사용 후 콘솔의 [solo_todo] cold start = …ms 줄 확인. 자세한 frame timing 은 flutter run --profile + DevTools Performance Overlay 사용.

Supabase 대시보드

Database → Tables → schema solo_todo → todos 에서 row 확인. Realtime 활성 시 변경 즉시 반영.

  • 콜드 스타트 1초 이하 확인
  • DevTools Performance 1 분 트레이스 — 평균 fps ≥ 60
  • macOS ↔ Android todo 변경이 5초 이내 양방향 동기화
