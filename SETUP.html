Solo Todo — 첫 실행 SETUP 가이드

앱 코드 외부에서 한 번 셋업이 필요한 항목. 위에서 아래로 순서대로 진행해 주십시오. 체크박스를 하나씩 채우면서 진행하시면 빠짐없이 처리됩니다.

1 환경변수 .env.local

저장소 루트의 .env.example 을 .env.local 로 복사 후 4 키를 채우십시오. .env.local 은 .gitignore 에 포함되어 있어 커밋되지 않습니다.

키발급처비고
SUPABASE_URL Supabase Dashboard → Project Settings → API 예: https://abc.supabase.co
SUPABASE_ANON_KEY 동일 JWT (eyJ…)
GOOGLE_OAUTH_CLIENT_ID_DESKTOP Google Cloud Console → APIs & Services → Credentials Type: Desktop. macOS 빌드용
GOOGLE_OAUTH_CLIENT_ID_ANDROID 동일 Type: Android, package + SHA-1 fingerprint 필요

실행 시 주입

flutter run --dart-define-from-file=.env.local
flutter build macos --release --dart-define-from-file=.env.local
flutter build apk --release --dart-define-from-file=.env.local
  • Supabase URL + Anon Key 입력
  • Google OAuth Desktop client id 입력 (Calendar 안 쓸 거면 비워둬도 OK)
  • Google OAuth Android client id 입력

2 Supabase 프로젝트 + RLS

  1. Supabase Dashboard 에서 새 프로젝트 생성 (region: ap-northeast-2 서울 권장).
  2. 좌측 메뉴 Authentication → Providers → Email 활성화. "Magic Link" 사용.
  3. Database → SQL Editor 에서 아래 SQL 그대로 실행:
-- todos 테이블 생성
create table if not exists public.todos (
  id              text primary key,
  user_id         uuid references auth.users(id) on delete cascade not null,
  title           text not null,
  category        text not null,
  due_at          timestamptz,
  done_at         timestamptz,
  created_at      timestamptz not null default now(),
  updated_at      timestamptz not null default now(),
  calendar_event_id text
);

-- RLS 활성
alter table public.todos enable row level security;

-- 본인 row 만 접근
create policy "user_read_own"   on public.todos for select  using (auth.uid() = user_id);
create policy "user_insert_own" on public.todos for insert  with check (auth.uid() = user_id);
create policy "user_update_own" on public.todos for update  using (auth.uid() = user_id);
create policy "user_delete_own" on public.todos for delete  using (auth.uid() = user_id);

-- 동기화 정렬용 인덱스
create index todos_user_updated on public.todos (user_id, updated_at desc);
  1. Database → Replication 에서 public.todos 의 Realtime 활성화. (Postgres logical replication)
  2. Authentication → URL Configuration → Redirect URLs 에 매직링크 콜백 URL 추가:
    • macOS: io.supabase.solo_todo://login-callback
    • Android: com.goldenplanet.solo_todo://login-callback
  • 프로젝트 생성
  • Email Magic Link 활성화
  • SQL 실행 (todos 테이블 + RLS)
  • Realtime 활성화
  • Redirect URLs 두 개 등록

3 Google Cloud (Calendar API)

  1. console.cloud.google.com 에서 새 프로젝트 생성.
  2. APIs & Services → Library → "Google Calendar API" Enable.
  3. OAuth consent screen 설정 (External + 본인 이메일).
  4. Credentials → + Create credentials → OAuth client ID:
    • Desktop app 타입 → client ID 발급 → GOOGLE_OAUTH_CLIENT_ID_DESKTOP 에 입력
    • Android 타입 → package: com.goldenplanet.solo_todo + SHA-1 fingerprint 
      cd android && ./gradlew signingReport
      출력 중 SHA1 값을 복사. GOOGLE_OAUTH_CLIENT_ID_ANDROID 에 입력.
  5. Scope 는 https://www.googleapis.com/auth/calendar.events 한 개. 코드가 자동 요청.
  • Calendar API Enable
  • OAuth consent screen 설정
  • Desktop client ID 발급 + .env.local 입력
  • Android client ID 발급 + .env.local 입력

4 macOS 빌드 (.app)

Xcode 풀 설치 + CocoaPods 가 필요합니다. 라이트한 Command Line Tools 만으론 부족합니다. 
# 1) Xcode 풀 설치 (Mac App Store) + 첫 실행
sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -runFirstLaunch

# 2) CocoaPods
sudo gem install cocoapods
# 또는 Homebrew: brew install cocoapods

# 3) 빌드
flutter pub get
pod install --project-directory=macos
flutter build macos --release --dart-define-from-file=.env.local

산출물: build/macos/Build/Products/Release/solo_todo.app

Entitlements (참고)

App Sandbox + Network Client 권한이 이미 macos/Runner/Release.entitlements 에 적용돼 있습니다. Supabase / Google Calendar outbound HTTPS 자동 동작.

  • Xcode 풀 설치 + xcode-select --switch
  • CocoaPods 설치
  • flutter build macos --release 성공
  • .app 더블클릭 후 sign-in 동작 확인
  • 메뉴바 (tray) 아이콘 노출 확인
  • Cmd+N 글로벌 단축키 동작 확인 (앱 백그라운드에서도)

  • flutter build apk --release 성공 (≈ 75초)

6 Deep Link (매직링크 콜백)

이메일 매직링크 클릭 후 OS 가 앱으로 되돌아오게 하려면 URL scheme 등록이 필요합니다.

macOS — macos/Runner/Info.plist

<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleURLSchemes</key>
    <array><string>io.supabase.solo_todo</string></array>
  </dict>
</array>

Android — android/app/src/main/AndroidManifest.xml

<activity ...>
  ...
  <intent-filter>
    <action android:name="android.intent.action.VIEW"/>
    <category android:name="android.intent.category.DEFAULT"/>
    <category android:name="android.intent.category.BROWSABLE"/>
    <data android:scheme="com.goldenplanet.solo_todo" android:host="login-callback"/>
  </intent-filter>
</activity>
  • macOS Info.plist 의 CFBundleURLTypes 등록
  • Android intent-filter 등록
  • Supabase Dashboard 의 Redirect URLs 와 일치하는지 확인

7 운영 메모 & 성능 확인

자주 보는 명령

flutter run -d macos --dart-define-from-file=.env.local

# 로컬 실행 (Android — 기기 선택 후)
flutter devices
flutter run -d <device-id> --dart-define-from-file=.env.local

# 검증 (커밋 직전)

dart format --output=none --set-exit-if-changed .
flutter test

60fps 트레이스 (디자인 점수 확인용)

release 빌드로 1 분 사용 후 콘솔의 [solo_todo] cold start = …ms 줄 확인. 자세한 frame timing 은 flutter run --profile + DevTools Performance Overlay 사용.

Supabase 대시보드

Database → Tables → todos 에서 row 확인. Realtime 활성 시 변경 즉시 반영.

  • 콜드 스타트 1초 이하 확인
  • DevTools Performance 1 분 트레이스 — 평균 fps ≥ 60
  • macOS ↔ Android todo 변경이 5초 이내 양방향 동기화
